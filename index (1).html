⚡ Quick Setup Guide — 100% Free
1

Create a free HiveMQ Cloud broker

Go to console.hivemq.cloud → Sign up free → Create a cluster → Note your broker URL, username, and password. The free tier supports 100 connections and unlimited messages.

2

Create a free Supabase project (optional — for history)

Go to supabase.com → New project → Copy your URL and anon key. Run this SQL to create the data table:

CREATE TABLE zyvon_data (
  id        BIGSERIAL PRIMARY KEY,
  device_id TEXT NOT NULL,
  pin       TEXT NOT NULL,
  value     JSONB,
  ts        TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX ON zyvon_data (device_id, pin, ts DESC);
-- Enable Row Level Security (optional but recommended)
ALTER TABLE zyvon_data ENABLE ROW LEVEL SECURITY;
3

Connect this dashboard

Click Connection in the sidebar and enter your HiveMQ credentials. Your device token acts as both the MQTT username/password and device identifier.

4

Flash your ESP32 / Raspberry Pi Pico W (MicroPython)

Copy zyvon.py to your device, then:

from zyvon import Zyvon
import machine, time

z = Zyvon(
    token="YOUR_DEVICE_TOKEN",
    broker="YOUR.s1.eu.hivemq.cloud",
    debug=True
)

# Send temperature every second
while True:
    temp = machine.ADC(4).read_u16() / 1000
    z.virtual_write("V1", temp)
    z.loop()
    time.sleep(1)
5

Use in a browser / Node.js app (JavaScript)

// Include zyvon.js, then:
const device = Zyvon.begin("YOUR_DEVICE_TOKEN", "YOUR.hivemq.cloud", {
    debug: true
});

device.on("connect", () => {
    // Stream a sensor value every 2s
    device.stream("V1", () => (Math.random() * 100).toFixed(2), 2000);
});

// React to dashboard controls
device.on("V2", (val) => {
    console.log("LED set to:", val);
});
6

Host Zyvon for free

Deploy this dashboard to GitHub Pages (push HTML to a repo → Settings → Pages) or Vercel (import repo → deploy) — both are completely free. No server needed, all communication goes directly browser ↔ HiveMQ via WebSockets.

MQTT Broker (HiveMQ Cloud)
From HiveMQ Cloud console — no wss:// prefix
8884 for WebSocket TLS
Supabase (optional history)
Supabase is optional. Without it, data is only kept in memory during this session. With it, all pin values are stored in your free Postgres database and you can query history.
API Reference

JavaScript — Browser / Node.js

// Initialise
const z = new Zyvon({ token, broker, debug: true });
// or shorthand:
const z = Zyvon.begin(token, broker);

// Send value to dashboard
z.virtualWrite("V1", 42);
z.virtualWrite("V2", "hello");
z.virtualWrite("V3", { temp: 23.5, hum: 61 });

// Receive from dashboard
z.on("V4", (val) => { /* LED state etc */ });
z.onVirtualWrite("V5", (val) => { });

// Auto-stream a sensor at interval
z.stream("V1", () => readSensor(), 1000);  // every 1s
z.stopStream("V1");

// Events
z.on("connect", () => { });
z.on("disconnect", () => { });
z.on("data", (pin, val) => { });  // all pins
z.on("error", (err) => { });

// Logging & metadata
z.logEvent("Booted", "info");
z.setProperty({ firmware: "1.0", uptime: 3600 });
z.disconnect();

MicroPython — ESP32 / Pico W

from zyvon import Zyvon

z = Zyvon(token="TOKEN", broker="host.hivemq.cloud", debug=True)

z.virtual_write("V1", 3.14)          # send
z.on_virtual_write("V2", lambda v: print(v))  # receive
z.log_event("Booted")
z.set_property({ "fw": "1.0" })

# In your main loop:
z.loop()          # non-blocking poll
# OR blocking:
z.run(100)         # polls every 100ms

MQTT Topic Schema

# Device → Dashboard (uplink)
zyvon/{device_id}/V1          ← sensor data
zyvon/{device_id}/_log        ← log events
zyvon/{device_id}/_props      ← device metadata
zyvon/{device_id}/status      ← online/offline LWT

# Dashboard → Device (downlink)
zyvon/{device_id}/V2/set      ← write to pin
zyvon/{device_id}/cmd         ← commands
